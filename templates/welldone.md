# 结束流程

## 更新 state.md

请在结束工作前更新 `.vibe/state.md`：

1. **标记已完成内容**
   - 列出本次工作完成的内容
   - 更新相关任务状态

2. **记录当前假设**
   - 记录当前工作中的假设
   - 标注待验证的内容

3. **列出未解决问题**
   - 记录遇到的问题
   - 记录需要后续处理的事项

4. **更新当前断点**
   - 记录当前工作进度
   - 为下次启动提供清晰的起点

---

## 执行示例

在 `.vibe/state.md` 中添加类似以下内容：

```
[YYYY-MM-DD HH:MM:SS] 本次工作完成：
- 完成 XXX 功能的实现
- 更新了 YYY 配置文件

[YYYY-MM-DD HH:MM:SS] 当前假设：
- 假设 ZZZ 模块的行为符合预期（待验证）

[YYYY-MM-DD HH:MM:SS] 未解决问题：
- AAA 问题需要进一步调研
- BBB 功能需要优化

[YYYY-MM-DD HH:MM:SS] 当前断点：
- 已完成到第 N 个任务
- 下一步需要处理 CCC
```
